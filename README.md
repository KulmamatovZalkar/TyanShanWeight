# TyanShanWeight

Приложение для автоматизации взвешивания грузовиков на заводе.
Интегрируется с AccountingDRF через вебхук.

## Возможности

- Чтение веса с электронных весов по RS-232 (COM-порт)
- Антидребезг (стабилизация показаний)
- Фиксация тары и брутто, автоматический расчет нетто
- Ручной режим ввода (без весов)
- Список ожидающих машин (зафиксировали тару, ждут брутто)
- Сохранение в локальную БД (SQLite)
- Отправка данных на сервер AccountingDRF (webhook)
- Автоподсказки водителей, госномеров, компаний с сервера
- Захват фото с IP-камер при взвешивании
- Современный темный интерфейс с горячими клавишами

## Установка

### Требования

- Windows 10/11
- Python 3.10+ (рекомендуется 3.11)
- COM-порт для весов (USB-RS232 адаптер)
- Сеть для доступа к серверу AccountingDRF

### Шаг 1: Установка из исходников

```bash
# Клонировать репозиторий
git clone <repo-url>
cd TyanShanWeight

# Создать виртуальное окружение
python -m venv venv
venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt
pip install opencv-python

# Запустить
python main.py
```

### Шаг 2: Сборка EXE (для установки без Python)

```bash
# Активировать виртуальное окружение
venv\Scripts\activate

# Установить PyInstaller
pip install pyinstaller

# Собрать
build.bat
```

Результат: `dist\TyanShanWeight\TyanShanWeight.exe`

Папку `dist\TyanShanWeight\` можно скопировать на целевой компьютер и запускать без Python.

## Настройка

### Первый запуск

При первом запуске автоматически создаются:
- `config.json` — конфигурация
- `weighings.db` — локальная база данных
- `logs/` — папка с логами

### Настройка весов (F5 → COM-порт)

1. Подключите весы через USB-RS232 адаптер
2. Откройте **Настройки** (F5)
3. Выберите COM-порт (например COM3, COM4)
4. Параметры: 9600 бод, 8 бит, без паритета, 1 стоп-бит
5. Нажмите **Тест подключения**
6. Сохраните

### Настройка камер (F5 → Камеры)

Поддерживаются HLS (.m3u8) и RTSP потоки:

```
https://stream.example.com/live/camera1.m3u8
rtsp://admin:password@192.168.1.100/stream1
```

Можно настроить до 4 камер. Фото снимаются автоматически при фиксации тары и брутто.

## Подключение к серверу AccountingDRF

### Шаг 1: Настройка вебхука на сервере

На сервере AccountingDRF уже есть endpoint для приёма данных:

```
POST /api/v1/transactions/webhook/weighing/
```

Убедитесь что сервер доступен и endpoint работает:

```bash
curl -X POST https://your-server.com/api/v1/transactions/webhook/weighing/ \
  -H "Content-Type: application/json" \
  -d '{"id":1,"datetime":"2026-01-01T12:00:00","car_number":"TEST","tara":1000,"brutto":2000,"netto":1000,"fio":"Test","fraction":"Test"}'
```

Ответ должен быть: `{"status":"success","id":"..."}` или `{"status":"duplicate","id":"..."}`

### Шаг 2: Прописать URL в приложении

**Вариант А: Через настройки (F5 → API)**

1. Откройте Настройки (F5)
2. Перейдите на вкладку **API**
3. В поле URL введите: `https://your-server.com/api/v1/transactions/webhook/weighing/`
4. Таймаут: 30 сек
5. Сохраните

**Вариант Б: Через config.json**

Отредактируйте `config.json`:

```json
{
  "api": {
    "url": "https://your-server.com/api/v1/transactions/webhook/weighing/",
    "timeout": 30,
    "retry_count": 3,
    "retry_delay": 5
  }
}
```

### Шаг 3: Проверка

После настройки приложение автоматически:
1. При старте загружает справочники (водители, госномера, контрагенты, грузы) через `GET .../webhook/weighing/lookup/?type=all`
2. Показывает автоподсказки при вводе
3. При сохранении взвешивания отправляет данные на сервер
4. Если сервер недоступен — данные сохраняются локально и отправляются позже (retry: 5с, 30с, 60с, 5мин)

### Формат данных (webhook)

Приложение отправляет POST JSON:

```json
{
  "id": 123,
  "datetime": "2026-01-19T16:50:00",
  "car_number": "01KG123ABC",
  "tara": 8000.0,
  "brutto": 28000.0,
  "netto": 20000.0,
  "fio": "Иванов Петр Сергеевич",
  "fraction": "Щебень 5-20",
  "counterparty_id": "uuid-контрагента",
  "notes": "Заметка оператора",
  "photos_tara_base64": ["base64-encoded-jpeg..."],
  "photos_brutto_base64": ["base64-encoded-jpeg..."]
}
```

### Справочники (lookup API)

Приложение загружает справочники при старте:

```
GET /api/v1/transactions/webhook/weighing/lookup/?type=all
```

Ответ:

```json
{
  "counterparties": [{"id": "uuid", "title": "Компания"}],
  "fractions": [{"id": "uuid", "title": "Щебень 5-20"}],
  "drivers": [{"id": "uuid", "name": "Иванов П.С.", "counterparty_id": "uuid", "counterparty": "Компания"}],
  "cars": [{"id": "uuid", "number": "01KG123ABC", "brand": "КАМАЗ", "counterparty_id": "uuid", "counterparty": "Компания"}]
}
```

Справочники управляются через веб-панель AccountingDRF:
- **Водители**: `/admin/drivers`
- **Транспорт**: `/admin/vehicles`
- **Контрагенты**: `/admin/counterparties`

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| F1 | Зафиксировать тару (сохраняет как ожидающую) |
| F2 | Зафиксировать брутто |
| F3 | Сохранить полное взвешивание |
| Esc | Сброс (очистить форму) |
| F5 | Настройки |

## Рабочий процесс

### Заезд (фиксация тары)

1. Грузовик заезжает на весы
2. Оператор вводит **госномер** (автоподсказка из справочника)
3. Выбирает **компанию** и **водителя**
4. Нажимает **F1** — тара фиксируется, машина в "ожидающих"
5. Грузовик уезжает загружаться

### Выезд (фиксация брутто)

1. Грузовик возвращается на весы
2. Оператор вводит **госномер** — система автоматически находит ожидающую запись и подгружает тару
3. Выбирает **груз** (фракцию)
4. Нажимает **F2** — брутто фиксируется, нетто рассчитывается
5. Нажимает **F3** — данные сохраняются и отправляются на сервер

### Ручной режим

Для ввода без весов нажмите кнопку **РЕЖИМ: АВТО** → переключится на **РЕЖИМ: РУЧНОЙ**.
В ручном режиме тару и брутто вводите вручную в поля.
